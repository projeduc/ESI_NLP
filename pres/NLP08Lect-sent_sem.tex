\section{Semantic roles}
%===================================================================================

\begin{frame}
	\frametitle{\insertshortsubtitle}
	\framesubtitle{\insertsection}
	
	\begin{itemize}
		\item \optword{Grammatical / Syntactic roles:} define how words function in a sentence.
		\begin{itemize}
			\item Examples: \expword{Subject, Object, Indirect Object, Modifier}
			\item Captured by dependency or constituency parses
		\end{itemize}
		
		\item \optword{Semantic roles:} define the underlying participant functions in events or actions.
		\begin{itemize}
			\item Examples: \expword{Agent, Patient, Experiencer, Theme, Instrument}
			\item Abstract away from surface syntax
			\item Provide \textbf{meaning-level understanding} for tasks like SRL, AMR, question answering
		\end{itemize}
		
		\item Example:
		\begin{itemize}
			\item Syntax: John (subject) was hit by the ball (object)
			\item Semantics: ball = Agent, John = Patient
		\end{itemize}
	\end{itemize}
\end{frame}

\subsection{FrameNet}

\begin{frame}
	\frametitle{\insertshortsubtitle: \insertsection}
	\framesubtitle{\insertsubsection}

	\begin{minipage}{.75\textwidth}
		
		\begin{itemize}
			\item {\scriptsize \url{https://framenet.icsi.berkeley.edu/fndrupal/}}
			\item {\scriptsize \url{https://www.nltk.org/howto/framenet.html}}
			\item FrameNet is a lexical resource used to represent meaning through frames.
			\item Based on the \keyword{Frame semantics} theory of \keyword{Fillmore}.
			\item Examples are manually annotated with semantic roles (frame elements).
			\item Each frame contains multiple semantic roles.
		\end{itemize}
	\end{minipage}
	\begin{minipage}{.23\textwidth}
		\hgraphpage{frameNet-logo.jpg}
	\end{minipage}
	
	\begin{exampleblock}{Example sentences evoking the same frame}
		The price of petrol increased.
		
		The price of petrol rose.
		
		There has been a rise in the price of petrol.
		
		\small{\textit{(All sentences evoke the frame ``Increase\_price''.)}}
	\end{exampleblock}

\end{frame}

\begin{frame}
	\frametitle{\insertshortsubtitle: \insertsection}
	\framesubtitle{\insertsubsection: Structure}
	
	\begin{itemize}
		\item \optword{Frame}: a schematic representation of a situation.
		\begin{itemize}
			\item Includes a name, definition, semantic type, frame elements, lexical units, examples, and relations to other frames.
		\end{itemize}
		
		\item \optword{Frame Element (FE)}: a semantic role specific to a frame. 
		\begin{itemize}
			\item Describes a participant, property, or attribute of the frame.
			\item Two types: central (\keyword{Core}) and peripheral (\keyword{Non-Core}).
		\end{itemize}
		
		\item \optword{Lexical Units (LU)}: lemmas associated with their part-of-speech (POS).
		\begin{itemize}
			\item A lexical unit evokes the frame.
		\end{itemize}
		
		\item \optword{Relations}: relations between frames.
		\begin{itemize}
			\item For example, \expword{inheritance}.
		\end{itemize}
	\end{itemize}
	
\end{frame}

\subsection{PropBank}

\begin{frame}
	\frametitle{\insertshortsubtitle: \insertsection}
	\framesubtitle{\insertsubsection}
	
	\begin{itemize}
		\item {\scriptsize \url{https://propbank.github.io/}}
		\item {\scriptsize \url{https://www.nltk.org/howto/propbank.html}}
		\item A corpus annotated with predicate--argument structures.
		\item Uses a small set of general semantic roles (ARG0--ARG5).
		\begin{itemize}
			\item \optword{ARG0}: typically the agent (doer of the action)
			\item \optword{ARG1}: typically the patient or theme
			\item Other arguments (ARG2--ARG5) depend on the predicate
		\end{itemize}
	\end{itemize}
	
\end{frame}

\begin{frame}
	\frametitle{\insertshortsubtitle: \insertsection}
	\framesubtitle{\insertsubsection: Structure}
	
	\begin{itemize}
		\item \optword{Roleset ID}: Each verb sense is associated with a roleset identifier.
		\begin{itemize}
			\item E.g., \expword{know.01: be cognizant of, realize; know.02: be familiar with, have experienced}
		\end{itemize}
		
		\item \optword{Roles}: Each verb sense has a set of possible arguments.
		\begin{itemize}
			\item \optword{Arg0}: typically agent-like
			\item \optword{Arg1}: typically patient-like or theme
			\item \optword{Arg2}: often expresses a beneficiary, instrument, attribute, or end state
			\item \optword{Arg3}: often expresses a starting point, beneficiary, instrument, or attribute
			\item \optword{Arg4}: typically endpoint
			\item Arg2--Arg5 are verb-specific; less consistently defined across predicates
		\end{itemize}
		
		\item \optword{Modifiers}: marked by \keyword{ArgM}  
		\begin{itemize}
			\item \optword{ArgM-TMP}: temporal (when)
			\item \optword{ArgM-LOC}: locative (where)
			\item \optword{ArgM-MNR}: manner (how)
			\item ...
		\end{itemize}
		
		\item \optword{Annotated examples}: illustrate predicate--argument structures
	\end{itemize}
	
\end{frame}

\subsection{Feature-based}

\begin{frame}
	\frametitle{\insertshortsubtitle: \insertsection}
	\framesubtitle{\insertsubsection}
	
	\begin{itemize}
		\item \optword{Idea}: traverse the parse tree and classify syntactic constituents.
		\begin{itemize}
			\item \optword{Input}: features extracted from the node (phrase) and the predicate.
			\item \optword{Output}: labels correspond to FrameNet or PropBank roles, plus \keyword{None} for non-arguments.
		\end{itemize} 
		
		\item The classification process can be optimized through:
		\begin{enumerate}
			\item \optword{Pruning}: eliminate candidate constituents using heuristics.
			\item \optword{Identification}: classify nodes as \keyword{Argument} or \keyword{None}.

		\begin{itemize}

\begin{frame}
	\frametitle{\insertshortsubtitle: \insertsection}
	\framesubtitle{Why parse into logical forms?}
	\scriptsize
	Even with PLMs, parsing into FOL or other logical forms remains useful:
	
	\begin{itemize}
		\item \textbf{Explicit reasoning}: Logical forms allow deduction, multi-step inference, and consistency checks.
		PLMs are great at pattern recognition and generating text, but they do not explicitly reason over entities, relations, and rules.
		
		\item \textbf{Compositionality}: Captures how meanings combine systematically (e.g., quantifiers, nested clauses).
		PLMs often struggle with rare compositions or longer chains of reasoning, especially for unseen combinations.
		
		\item \textbf{Cross-domain generalization}: A single reasoning engine can handle multiple domains once sentences are mapped to logic.
		PLMs might need extensive retraining or prompting for new domains.
		
		\item \textbf{Explainability}: Logical forms are human-readable, verifiable, and interpretable.
		PLMs' reasoning is opaque (``black box''), making it harder to trust outputs for sensitive decisions.
		
		\item \textbf{Integration with symbolic systems}: Enables database queries, knowledge graph reasoning, and robotics control.
		
		
		\item \textbf{Benchmarking reasoning ability}: Provides ground truth for testing PLMs' logical reasoning and compositional generalization.
		
		\item \textbf{Neuro-symbolic approaches}: Combines PLMs' flexibility with rigorous symbolic reasoning.
	\end{itemize}
	
\end{frame}


\subsection{First order logic}

\begin{frame}
	\frametitle{\insertshortsubtitle: \insertsection}
	\framesubtitle{\insertsubsection}
	
	\begin{itemize}
		\item \optword{Term}: an expression representing an object.
		\begin{itemize}
			\item \optword{Constant}: a specific object (domain-specific).  
			E.g. \expword{Karim, ESI, Algeria}
			
			\item \optword{Function}: returns an object based on one or more input terms mapping terms to a term in the domain.  
			E.g. \expword{LocationOf(ESI)}
			
			\item \optword{Variable}: a symbol representing an unspecified object.
			E.g. \expword{x, y, z}
		\end{itemize}
		
		\item \optword{Predicate}: a relation over terms.
		E.g. \expword{University(ESI), TeacherAt(Karim, ESI)}
		
		\item \optword{Connective}: logical operators;
		AND ($\wedge$), OR ($\vee$), NOT ($\neg$), IMPLY ($\rightarrow$), EQUIVALENT ($\Leftrightarrow$).
		E.g. \expword{TeacherAt(Karim, ESI) $\wedge$ University(ESI)}
		
		\item \optword{Quantifier}: expresses existence or universality;
		EXISTS ($\exists$) and FOR ALL ($\forall$)  
		
		\expword{Natural sentence: "I eat at a restaurant near ESI."} 
		 
		\expword{FOL: $\exists x \, (Restaurant(x) \wedge Near(LocationOf(x), LocationOf(ESI)) \wedge Eat(Interlocutor, x))$}
	\end{itemize}
	
\end{frame}

\begin{frame}
	\frametitle{\insertshortsubtitle: \insertsection}
	\framesubtitle{\insertsubsection: Components in FOL systems}
	
	\begin{itemize}
		\item \textbf{Variable components:} can differ between FOL systems
		\begin{itemize}
			\item \optword{Constants}: specific objects present in the domain
			\item \optword{Functions}: used to generate objects not represented as constants
			\item \optword{Predicates}: depend on which relations or properties the system designer chooses to represent
		\end{itemize}
		
		\item \textbf{Fixed components:} common to all FOL systems
		\begin{itemize}
			\item \optword{Variables}: generic placeholders
			\item \optword{Connectives}: AND, OR, NOT, IMPLY, EQUIVALENT
			\item \optword{Quantifiers}: EXISTS ($\exists$), FOR ALL ($\forall$)
		\end{itemize}
		
	\end{itemize}
	
\end{frame}


\begin{frame}
	\frametitle{\insertshortsubtitle: \insertsection}
	\framesubtitle{\insertsubsection: Expressiveness of FOL systems}
	
	\begin{itemize}
		\item The choice of \textbf{constants} and \textbf{functions} depends on the domain:
		\begin{itemize}
			\item If the domain lacks specific constants, \textbf{functions} can generate objects.
			\item Example: \expword{LocationOfESI} vs. \expword{LocationOf(ESI)}.
		\end{itemize}
		
		\item The \textbf{arity of predicates} can limit expressivity:
		\begin{itemize}
			\item Standard FOL predicates have fixed arity; e.g., \expword{Eat(Person,Food)}.
			\item Different syntactic forms (I ate, I ate lunch, I ate at my desk, etc.) may require different numbers of arguments.
		\end{itemize}
		
		\item \textbf{Neo-Davidsonian event representation} addresses this:
		\begin{itemize}
			\item Introduce an \textbf{event variable} for each event: \expword{$\exists e\ Eating(e)$}.
			\item Attach roles as separate predicates linked to the event:  \\
			\expword{Eater(e,Speaker) $\wedge$ Eaten(e,TurkeySandwich) $\wedge$ Location(e,Desk)}
			\item Benefits:
			\begin{itemize}
				\item No need to fix the number of arguments per predicate.
				\item Can represent additional roles (Time, Location, Instrument, etc.) flexibly.
				\item Logical connections are preserved without extra inference.
			\end{itemize}
		\end{itemize}
		
%		\item Choosing constants, functions, and predicates carefully remains essential for accurate and efficient semantic representation.
	\end{itemize}
	
\end{frame}

\subsection{Semantic parsing}

\begin{frame}
	\frametitle{\insertshortsubtitle: \insertsection}
	\framesubtitle{\insertsubsection}
	
	\begin{itemize}
		\item \textbf{Semantic parsing}: the process of mapping natural language sentences to a formal meaning representation.
		\begin{itemize}
			\item Converts text into logical forms, AMR graphs, or structured queries.
			\item Enables reasoning, question answering, database querying, and NLU tasks.
		\end{itemize}
		
		\item \textbf{Challenges}:
		\begin{itemize}
			\item Ambiguity in natural language
			\item Variable numbers of arguments, complex sentence structure
			\item Maintaining compositionality for larger sentences
		\end{itemize}
		
		\item \textbf{Role of $\lambda$-calculus}:
		\begin{itemize}
			\item Provides a formal way to express functions over entities and events
			\item Allows composition of sentence meanings from smaller units (words or phrases)
			\item Facilitates variable binding and reduction, simplifying logical representation
		\end{itemize}
	\end{itemize}
	
	
\end{frame}

\begin{frame}
	\frametitle{\insertshortsubtitle: \insertsection}
	\framesubtitle{\insertsubsection: Lambda notation}
	
	\begin{itemize}
		\item \keyword{$\lambda$-Expression}
		\begin{itemize}
			\item Represents an anonymous function
			\item \keyword{$\lambda$x.P(x)} an anonymous function that takes an argument $x$ and returns $P(x)$
			\item Example: \expword{$\lambda$x.USE(x, BERT)} to express the clause \expword{x uses BERT}
		\end{itemize}
		
		\item \keyword{$\lambda$-Reduction}
		\begin{itemize}
			\item Substitute a variable by an expression
			\item \keyword{$ \phi(\psi) $} means ``apply function $\phi$ to argument $\psi$''.
			\item Example: \expword{$\lambda$x.USE(x, BERT)\ KARIM = USE(KARIM, BERT)}
		\end{itemize}
		
		\item For fun: \url{http://davidmarino.nfshost.com/lambdalinguist/index.html}
	\end{itemize}
	
\end{frame}

	\frametitle{\insertshortsubtitle: \insertsection}
	\framesubtitle{\insertsubsection: Neural approaches to logical form generation}
	
	\begin{itemize}
		\item Recent work uses pretrained LLMs for semantic parsing.
		\item They can map natural language to logical forms directly.
		\item \textbf{Advantages}:
		\begin{itemize}

			\item Flexible handling of quantifiers, events, and argument structure.
		\end{itemize}

		\begin{itemize}
			\item \expword{Input: ``Every student uses BERT''}
			\item \expword{Output (LogicLLaMA/T5): $\forall x\ (Student(x) \rightarrow USE(x, BERT))$}
		\end{itemize}
		\item These methods can also produce FOL, lambda calculus, or AMR representations.
